```markdown
# Проект: Обработка заказов и проверка соответствия ТК

Этот проект предназначен для обработки заказов, извлечённых из Excel-файлов, и проверки их соответствия техническим условиям (ТК). Проект реализован с разделением логики на несколько модулей для повышения читаемости, сопровождения и тестируемости кода.

## Архитектура проекта

Проект структурирован следующим образом:

```
project/
├── README.md                 # Описание проекта и инструкции по запуску
├── requirements.txt          # Список зависимостей проекта (например, openpyxl, numpy, pandas и др.)
├── setup.py                  # Скрипт установки/пакетирования проекта (опционально)
├── config/                   # Папка для конфигурационных файлов
│   ├── app_config.txt
│   ├── key_value.txt
│   ├── service_config.txt
│   └── status_tk.txt
├── data/                     # Папка для входных файлов, тестовых данных и результатов работы
│   ├── test_zak.xlsx
│   
├── logs/                     # Папка для логов работы приложения
│   
├── src/                      # Исходный код проекта
│   ├── __init__.py
│   ├── main.py               # Точка входа в приложение
│   ├── config_parser.py      # Модуль для обработки конфигурационных файлов
│   ├── excel_processor.py    # Модуль для работы с Excel и извлечения данных
│   ├── validators/           # Папка для модулей проверки соответствия заказов техническим условиям
│   └── utils.py              # Вспомогательные утилиты и функции

```

## Установка

1. **Клонирование репозитория:**

   ```bash
   git clone https://github.com/username/my-project.git     --ЗАМЕНИТЬ НА РЕАЛЬНОЕ ИМЯ ПРОЕКТА--
   cd my-project
   ```

2. **Создание виртуального окружения (venv):**

   ```bash
   python -m venv venv
   ```

3. **Активация виртуального окружения:**

   - На Windows:
     ```bash
     venv\Scripts\activate
     ```
   - На macOS/Linux:
     ```bash
     source venv/bin/activate
     ```

4. **Установка зависимостей:**

   ```bash
   pip install -r requirements.txt
   ```

## Запуск проекта

После установки зависимостей вы можете запустить основное приложение:

```bash
python src/main.py
```

Процесс обработки заказа будет выполняться с использованием конфигурационных файлов из папки **config/** и входных данных из папки **data/**. Результаты работы выводятся как в консоль, так и записываются в соответствующие файлы.

## Тестирование

Для запуска модульных тестов можно использовать [pytest](https://docs.pytest.org/):

```bash
pytest tests/
```

## Структура кода

- **src/config_parser.py:** Парсинг и обработка конфигурационных файлов, содержащих структуру обрабатываемых листов.
- **src/excel_processor.py:** Извлечение данных из Excel-файлов с использованием `openpyxl` (или `pandas` при необходимости) с последующей структуризацией данных.
- **src/validators.py:** Логика проверки заказов на соответствие техническим условиям (ТК). Здесь реализованы функции для валидации параметров заказа.
- **src/utils.py:** Общие утилиты (например, функция проверки диапазонов, конвертация файлов и т.д.).

## Дополнительная информация

- **Логи и результаты:**  
  Логи работы приложения сохраняются в папке **logs/**, а выходные файлы с результатами обработки – в **data/**.
  
- **Конфигурация:**  
  Все параметры и настройки вынесены в конфигурационные файлы, расположенные в **config/**, что облегчает настройку проекта без внесения изменений в исходный код.
